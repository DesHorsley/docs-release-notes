_January 28, 2018_

This document describes new features and enhancements, known limitations, issues, and fixes in Anypoint Runtime Manager Agent, version 1.12.0.

== New Features and Enhancements

This release contains no new features or enhancements.

== Changes in This Release

* Changed to connect to the new Runtime Manager Publicly certified endpoints for Mule Manager and Auth Proxy
* Now the Mule Agent API returns bad request (400) instead of internal server error (500) when enabling a service with null log configuration

== Fixed Issues

This release contains no issue fixes.

== Known Limitations and Workarounds

There are no known limitations or workarounds in this release.

== Upgrade Requirements

* Upgrade procedure will result in the use of 2 new endpoints for communicating with Anypoint Runtime Manager. Before doing any upgrade, please ensure there is no networking constraint accessing the following FQDNs on port 443:
    * ws://runtime-manager.anypoint.mulesoft.com, ws://runtime-manager.eu1.anypoint.mulesoft.com
    * https://data-authenticator.anypoint.mulesoft.com, https://data-authenticator.eu1.anypoint.mulesoft.com
